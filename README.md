# readme-is-all-you-need (業務ナビゲーター)

> **"勝手に変なもの入れて怒られない制約下でも、業務を整理する。"**
> インターネット・サーバー・インストール・ビルド不要。極限まで依存を削ぎ落とした、オフライン完結型業務検索・閲覧ツール。
> 中の人はファイルのディレクトリ管理のついでで設計しました。
> マニュアルの電子化などの足掛かりにご利用いただけます。

## プロジェクトの目的
バイト先のちょっとしたストレスの蓄積が嫌になってきたのでスクリプト準備する。労働環境改善や
共有PCでのファイル乱立、手順書の散逸、二重ダウンロード。これらを解決するために、最も導入ハードルが低く、かつモダンな操作性を持つツールを目指して開発
マニュアルの入った場所を探してて対応遅れてクレームとか馬鹿すぎるので検索ツールはあってしかるべきなんですよ、というか見つからないのはもはや何かのハラスメントでは？
ほぼなんも入れてない環境でも、`index.html` をダブルクリックするだけで即座に業務ナレッジベースが立ち上がります。

---

## 主な特徴

- **ゼロ・デマンド (Zero Dependency)**
  - 外部CDN、フレームワーク（React/Vueなど）、APIを一切不使用のはず(別にサンドボックスとかで開発したわけじゃない)。
  - HTML / CSS / Vanilla JS のみで構成。オフライン環境でも100%動作。
- **file:// プロトコル完全対応**
  - ブラウザのセキュリティ制限（CORS）に抵触しないよう、データをJSファイルとして読み込む設計。こいつを犠牲にすればだいぶ利便性上げられるはず。
- **LocalStorage による自動保存**
  - 追加・編集した内容はブラウザに即時保存。サーバーレスながら、快適なCRUD操作を実現。
  - サーバレスは普通に使いにくいのがポイント！
- **Fluent Design (Windows 11 準拠)**
  - Windows 11のUIガイドラインに準じたフラットで清潔感のあるデザイン。
  - 作成してからバ先のPCが11じゃないことに気づいたがもう遅い。使いながら調整する。
- **超高速インクリメンタルサーチ**
  - 入力と同時にDOMをフィルタリング。マニュアルやパスへ瞬時にアクセス。
- **管理者向けスクリプト同梱**
  - PowerShellによる自動バックアップと整合性チェック機能を備え、運用性も考慮。


## 使い方

1. 本リポジトリをクローン、またはZIPでダウンロードして展開します。
2. `index.html` をブラウザ（Edge / Chrome推奨）で開きます。
3. **以上です。**
4. LocalStrageに保存したろのクソ仕様で作ったので例えばネットワークフォルダとかに置いても共有されません。注意してください。制約がキツいんじゃ
複数端末からアクセスする場合、現状でも初期設定にペーストして都度リセットかければそれっぽいことは可能ですが、エンジニアと相談して同期できるよう修正してお使いください。
